ASN C Runtime Precondition (macro ASN_C_REQUIRE)

The ASN C Runtime Precondition mechanism (available through the macro ASN_C_REQUIRE) is part of ASN C Runtime assertion mechanism. Assertions are used extensively by the ASN C Runtime. Assertions are a way to embed formal specifications in the code and therefore a way to provide a clean and simple mechanism for checking assumptions at runtime (in debug build only). If during execution time an assertion fails then a diagnostic message (including the precise location) is reported. On Unix the diagnostic message is simply printed to the standard error. On Windows NT and Windows 2000 the ASN C Runtime assertion mechanism relies on the C runtime debug features (including the use of the _ASSERTE macro defined in crtdbg.h, see Visual C++ documentation for more information).



A precondition is a specific kind of assertion that must be used only to check the validity of the imput parameters of a C function. Ideally if the preconditions are all true the function must be able to fulfill its job. A precondition is like a contract between a caller of a function and the function. If all the terms of the contract are right, then the deal (i.e. successful execution of the function) can be made. An invalid precondition always denotes an in error in the caller code of a function.
